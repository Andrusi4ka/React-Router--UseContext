# Вступ до маршрутизації в React з використанням React Router

### 1. Ініціалізація проекту

- Створіть новий React проект за допомогою Vite:
```cmd
npm create vite@latest my-react-router-app -- --template react
```

- Перейдіть до каталогу проекту:
```cmd
cd my-react-router-app
```

- Встановіть залежності:
```cmd
npm install
```

- Встановіть бібліотеку react-router:
```cmd
npm install react-router
```

### 2. Створення компонентів

- У директорії `src/components` створіть три компоненти: `Home.jsx`, `About.jsx`, `Contact.jsx`.
- Кожен компонент повинен відображати унікальний текст, який ідентифікує його призначення (наприклад, "Це домашня сторінка", "Це сторінка про нас", "Це контактна сторінка").

### 3. Налаштування маршрутизації

- У файлі `src/App.jsx` імпортуйте та використовуйте `BrowserRouter` для ініціації маршрутизації.
- Налаштуйте маршрути за допомогою компонентів `Route`:
- / → Home
- /about → About
- /contact → Contact
- Використовуйте компонент Link для створення навігаційних посилань у головному меню.

### 4. Використання NavLink

- Замініть `Link` на `NavLink` у головному меню.
- Налаштуйте NavLink, щоб активне посилання виділялося стилем або класом.

### 5. Тестування проекту

- Запустіть проект:

```cmd
npm run dev
```

- Переконайтеся, що перехід між сторінками відбувається без перезавантаження браузера.

### 6. Деплоймент

- Розгорніть додаток на платформі хостингу (наприклад, Vercel або Netlify).

## Використані хуки

- `useState` — це хук, який дозволяє зберігати та змінювати стан (state) компонента.
Він повертає поточне значення стану та функцію для його оновлення, після виклику якої компонент перерендерюється.
Використовується для зберігання даних, що можуть змінюватися з часом (лічильник, форма, отримані з API дані тощо).

- `useEffect` — це хук, який дозволяє виконувати побічні дії в компоненті (запити до API, підписки, робота з DOM).
Він запускається після рендеру компонента і може виконуватись знову, коли змінюються залежності.
Також використовується для очищення ефектів (cleanup), наприклад при розмонтуванні компонента.

- `useContext` — це хук, який дозволяє отримувати дані з React Context без передачі пропсів через багато рівнів компонентів.
Він дає доступ до глобального стану або функцій, створених у Context Provider.
Використовується для теми, авторизації, мови, налаштувань або спільних даних у всьому додатку.

- `useParams` — це хук з React Router, який дозволяє отримати параметри з URL (наприклад id у /users/:id).
Він повертає об’єкт з усіма параметрами маршруту у вигляді рядків.
Використовується для завантаження або відображення даних на основі адреси сторінки.

- `Link` — це компонент з React Router, який використовується для навігації між сторінками без перезавантаження сторінки.
Він змінює URL і рендерить відповідний компонент, зберігаючи стан застосунку.
Використовується замість тега `<a>`, щоб навігація працювала швидко та коректно в SPA.

- `Route` — це компонент з React Router, який визначає, який компонент потрібно відобразити для конкретного URL-шляху.
Він порівнює поточний шлях у браузері з path і, якщо вони збігаються, рендерить вказаний компонент.
Використовується всередині Routes для побудови маршрутизації в React-додатку.

- `Routes` — це компонент з React Router, який обгортає всі `Route` і відповідає за вибір одного відповідного маршруту.
Він перевіряє поточний URL і рендерить лише той `Route`, шлях якого збігається.
Використовується для організації та керування всією маршрутизацією в React-додатку.

- `NavLink` — це компонент з React Router для навігації, який працює як Link, але додатково визначає активний маршрут.
Він автоматично додає клас або стиль активному посиланню, якщо його шлях збігається з поточним URL.
Використовується для меню навігації, щоб показати користувачу, на якій сторінці він зараз знаходиться.

- `BrowserRouter` — це компонент з React Router, який підключає маршрутизацію на основі history API браузера.
Він дозволяє змінювати URL без перезавантаження сторінки та керує навігацією в SPA.
Використовується як обгортка навколо всього React-додатку, де потрібна маршрутизація.

- `createContext` — це функція React, яка створює Context для спільного зберігання даних у додатку.
Вона дозволяє передавати значення (state, функції, налаштування) глибоко по дереву компонентів без пропсів.
Зазвичай використовується разом із Context.Provider та хуком useContext.

- `useNavigate` — це хук з React Router, який дозволяє програмно змінювати маршрут.
Він використовується, коли потрібно перейти на іншу сторінку після дії користувача (клік, submit форми, успішний запит).
Працює як навігація без перезавантаження сторінки, так само як Link.
